## 📖 Overview

**Trade Opportunities API** accepts an Indian market sector name and returns a structured **Markdown report** covering:

- 📊 Market Overview
- 📈 Growth Trends
- 🚢 Export / Import Opportunities
- 🏛️ Government Schemes & Policies
- 💰 FDI & Investment Outlook
- ⚠️ Industry Challenges

Reports are generated by fetching **live search snippets** from DuckDuckGo across 6 targeted query dimensions, then synthesising them with the **Groq LLaMA 3.3-70B** language model. Results are cached for **30 minutes** to avoid redundant API calls.

## 🚀 Quick Start

### 1. Clone & create virtual environment

```bash
git clone https://github.com/<your-username>/trade-api.git
cd trade-api
python -m venv .venv

# Windows
.venv\Scripts\activate

# macOS / Linux
source .venv/bin/activate
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Configure environment variables

Create a `.env` file in the project root:

```env
GROQ_API_KEY=your_groq_api_key_here
TRADE_API_KEY=your_custom_api_key_here
PORT=8000
```

| Variable | Required | Description |
|---|---|---|
| `GROQ_API_KEY` | ✅ Yes | Get one free at [console.groq.com](https://console.groq.com/) |
| `TRADE_API_KEY` | Optional | Defaults to `trade-secret-key-2024` |
| `PORT` | Optional | Defaults to `8000` |

### 4. Run locally

```bash
python run.py
```

The server starts at `http://localhost:8000`.

- **Swagger UI** → `http://localhost:8000/docs`
- **ReDoc** → `http://localhost:8000/redoc`

### `GET /api/v1/analyze/{sector}`

Generate a trade opportunities report for an Indian market sector.

**Path Parameter**

| Parameter | Type | Description |
|---|---|---|
| `sector` | `string` | Sector name (2–60 chars, letters/numbers/spaces/hyphens) |

**Example Request**

```bash
# Production
curl -X GET "https://trade-api-g24o.onrender.com/api/v1/analyze/pharmaceuticals" \
     -H "X-API-Key: trade-secret-key-2024"

# Local
curl -X GET "http://localhost:8000/api/v1/analyze/pharmaceuticals" \
     -H "X-API-Key: trade-secret-key-2024"
```

**Example Response** `200 OK`

```json
{
  "sector": "pharmaceuticals",
  "session_id": "a1b2c3d4-e5f6-...",
  "generated_at": "2025-04-18T07:10:00+00:00",
  "report_markdown": "# Pharmaceuticals Sector — Trade Opportunities Report\n\n## Market Overview\n...",
  "sources_used": 24
}
```

**Error Responses**

| Status | Code | Description |
|---|---|---|
| `400` | `BAD_REQUEST` | Invalid sector name |
| `401` | `UNAUTHORIZED` | Missing API key |
| `403` | `FORBIDDEN` | Invalid API key |
| `429` | `RATE_LIMITED` | Too many requests; check `Retry-After` header |
| `503` | `SERVICE_UNAVAILABLE` | Groq AI service error |

## ☁️ Deploy to Render

This API is **live on Render**: https://trade-api-g24o.onrender.com

- 📚 **Swagger UI** → https://trade-api-g24o.onrender.com/docs
- 📖 **ReDoc** → https://trade-api-g24o.onrender.com/redoc

To deploy your own instance:

1. Fork this repository.
2. Go to [render.com](https://render.com/) → **New Web Service** → connect your fork.
3. Render will detect `render.yaml` automatically.
4. Set the `GROQ_API_KEY` environment variable in the Render dashboard (or update `render.yaml`).
5. Deploy!

> [!NOTE]
> Render free-tier services spin down after inactivity. The first request after a cold start may take ~30 seconds.

```yaml
# render.yaml (summary)
services:
  - type: web
    name: trade-api
    env: python
    buildCommand: pip install -r requirements.txt
    startCommand: uvicorn app.main:app --host 0.0.0.0 --port $PORT
```

## 🧪 Development Tips

**Run with auto-reload:**
```bash
python run.py   # reload=True is set by default in run.py
```

**Override rate limits for local testing:**  
Edit `app/middleware/rate_limiter.py`:
```python
MAX_REQUESTS = 100
ANALYZE_MAX_REQUESTS = 50
```

**Inspect logs:**  
Logs are printed to stdout in the format:
```
2025-04-18 07:10:00,123 | INFO | app.routers.analyze | [a1b2c3d4] Analyze request for sector: 'pharmaceuticals'
```
